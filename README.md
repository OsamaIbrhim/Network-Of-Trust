# Network Of Trust — powered dy Blockchain

**NoT v1.0 Stable Baseline**

A Hardhat-based Ethereum NoT with an Express read API, React admin dashboard, and MetaMask for on-chain writes. Local development targets **Ganache** on chain ID **1337**.

## Prerequisites

- [Node.js](https://nodejs.org/) 18+
- [Ganache](https://trufflesuite.com/ganache/) (GUI or CLI) listening on `http://127.0.0.1:7545`, chain ID `1337`
- [MetaMask](https://metamask.io/) configured with a custom network:
  - RPC URL: `http://127.0.0.1:7545`
  - Chain ID: `1337`
  - Currency: ETH

## Project layout

| Path | Purpose |
|------|---------|
| `contracts/` | Solidity source (source of truth for business logic) |
| `artifacts/contracts/` | Hardhat compile output — **ABI source of truth** (backend + frontend read directly) |
| `abis/` | Optional export from `npm run sync-abis` (external tooling only) |
| `scripts/deploy.js` | Deploy all contracts + wire validator + write `deployments/ganache.json` |
| `scripts/sync-abis.js` | Copy ABIs from artifacts into `abis/` |
| `backend/` | Express read-only API proxy |
| `frontend/` | React + Vite admin dashboard |

## Install

From the repository root:

```bash
npm install
npm install --prefix backend
npm install --prefix frontend
```

## Run from scratch

### 1. Start Ganache

Ensure Ganache is running at `http://127.0.0.1:7545` with chain ID `1337`.

### 2. Compile contracts and sync ABIs

```bash
npm run compile
```

This runs `hardhat compile` and then `npm run sync-abis`, which writes JSON ABIs to `abis/`.

### 3. Run tests

```bash
npm test
```

### 4. Deploy contracts

```bash
npm run deploy
```

Writes `deployments/ganache.json` with contract addresses and syncs ABIs.

### 5. Start backend

```bash
npm run backend:dev
```

Backend listens on `http://localhost:3000`.

Optional: copy `backend/.env` and set `RPC_URL=http://127.0.0.1:7545` and `CHAIN_ID=1337` if needed.

### 6. Start frontend

In a second terminal:

```bash
npm run frontend:dev
```

Open `http://localhost:5173`. Connect MetaMask to Ganache (chain 1337).

## Health check

- Backend: `GET http://localhost:3000/health`
- Frontend proxies `/health` and `/api` to the backend during development.

## ABI workflow

1. **Source of truth:** `artifacts/contracts/<Contract>.sol/<Contract>.json` (generated by Hardhat)
2. **Runtime consumers:** backend and frontend read ABIs directly from artifacts
3. **Optional export:** `npm run sync-abis` copies ABIs into `abis/` for external tooling

After changing Solidity contracts:

```bash
npm run compile
```

Do not maintain separate hand-edited ABI files in backend or frontend.

## Deployment addresses

After deploy, addresses are read from `deployments/ganache.json` by:

- `backend/src/config/contracts.ts`
- `frontend/src/web3/contracts.ts`

Redeploy whenever Ganache is reset so addresses stay in sync.

## Scripts reference

| Command | Description |
|---------|-------------|
| `npm run compile` | Compile contracts + sync ABIs |
| `npm run sync-abis` | Copy ABIs from artifacts to `abis/` |
| `npm test` | Hardhat integration tests |
| `npm run deploy` | Deploy to Ganache + sync ABIs |
| `npm run backend:dev` | Start Express API (ts-node) |
| `npm run backend:build` | Compile backend TypeScript |
| `npm run frontend:dev` | Start Vite dev server |
| `npm run frontend:build` | Production frontend build |

## Baseline scope notes

- **Admin dashboard** (`/`, `/identity`, `/certificates`, `/health`) is the primary wired UI.
- **Institution / Student / Employer** routes are placeholders in this baseline.
- On-chain writes use MetaMask against deployed contract ABIs.
- Backend exposes read-only endpoints mapped to existing contract view functions only.

## Troubleshooting

| Issue | Action |
|-------|--------|
| `Deployment file not found` | Run `npm run deploy` |
| `ABI not found` | Run `npm run compile` or `npm run sync-abis` |
| MetaMask wrong network | Switch to Ganache chain ID 1337 |
| Backend 503 on `/health` | Ensure Ganache is running and addresses in `deployments/ganache.json` are valid |
